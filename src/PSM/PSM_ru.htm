Соответствие оценок склонности

Эта процедура сопоставляет записи наблюдений со схожими записями контрольных значений, содержащимися в едином наборе данных. Сначала она запускает логистическую регрессию с переменной группы наблюдений/контрольных значений в качестве зависимой переменной. Затем она выбирает сопоставление для каждой группы контрольных значений на основе значений оценок склонности по логистической регрессии. Эти значения оценивают вероятность принадлежности к группе наблюдений.

Процедура генерирует и активирует новый набор данных наблюдений и сопоставленных контрольных значений.

Индикатор группы Выберите переменную, определяющую, находится ли запись в группе наблюдений или группе контрольных значений. Значение переменной 1 указывает на наблюдение, а значение 0 - на контрольное значение.

Предикторы Выберите переменные для использования в логистической регрессии для моделирования принадлежности к группе наблюдений/контрольных значений.

Имя для переменной склонности Введите имя переменной для хранения оценок (склонности), полученных по логистической регрессии. Эта переменная будет использоваться для проведения сопоставлений.

• Имя переменной не может уже использоваться.

Допуск сопоставления Задайте допуск для оценки сопоставления наблюдений и контрольных значений. Контрольное значение сопоставимо с наблюдением, если абсолютное значение разности в оценках склонности меньше или равно этому значению. Значение 0 означает только точные сопоставления, а значение 1 означает, что можно сопоставить любое контрольное значение с любым наблюдением. Меньшие значения генерируют более точные сопоставления, но могут увеличить число несопоставленных наблюдений.

• Выходные таблицы, генерируемые этой процедурой, показывают распределение числа приемлемых сопоставлений с учетом допуска и могут оказаться полезными при выборе значения допуска.

ID наблюдения Задайте переменную идентификатора наблюдения. Значение ID наблюдения для сопоставленного контрольного значения будет храниться с каждой записью наблюдения.

Имя переменной ID сопоставления Задайте имя для переменной, которая будет хранить ID выбранной записи контрольного значения. Если не удастся сгенерировать сопоставление, значение переменной будет системным значением отсутствия.

• Имя переменной не может уже использоваться.

Имя выходного набора данных Задайте имя для создаваемого набора данных. В этом наборе данных будут храниться все наблюдения и контрольные значения, которые были с ними сопоставлены.

• Имя набора данных не может уже использоваться.

• Вывод представляет собой набор данных, а не постоянный файл. Этот набор данных нужно сохранить, чтобы создать фактический файл.

Параметры

Переменная для количества подходящих наблюдений Дополнительно можно ввести имя переменной для записи размера пула контрольных значений (то есть количества подходящих наблюдений) для каждой записи наблюдения. Это число контрольных значений, соответствующих критериям сопоставления после удаления совпадений для предыдущих контрольных значений (в случае отбора без возвращения).

• Имя переменной не может уже использоваться

Отбор Наблюдения контрольных значений могут быть построены с возвращением или без него.

Отдать приоритет точным соответствиям Включите этот переключатель, чтобы алгоритм сначала пробовал проводить точное сопоставление, а затем нечеткое.

Максимизировать производительность выполнения Включите этот переключатель, чтобы максимизировать производительность при выборе сопоставлений (максимальная скорость и минимальное использование памяти). Для больших наборов данных этот переключатель должен быть включен. Если он используется, немного увеличивается вероятность, что не найдется наблюдение для сопоставления с контрольным значением.

• Нельзя задать сразу и точный приоритет, и минимизацию памяти.

Рандомизировать порядок наблюдений при построении сопоставлений Включите этот переключатель для построения сопоставлений записей наблюдений в случайном порядке. В противном случае записи наблюдений будут обрабатываться в порядке, соответствующем их порядку в файле. Это может привести к выбору в дальнейшем меньшего числа контрольных значений в файле. Если у записей наблюдений будет систематический порядок, при обработке в порядке файла может быть введено некоторое смещение.

• Если включена опция максимизации производительности выполнения, выбор этого параметра ни на что не влияет.

Начальное значение генератора случайных чисел Если вы хотите получить возможность точного воспроизведения сопоставления для будущего запуска с теми же самыми данными, можно задать начальное значение для случайных чисел, используемых при отборе из набора случайным образом. Это начальное значение не связано с общим начальным значением генератора случайных чисел SPSS Statistics.

Дополнительные возможности

Это диалоговое окно генерирует синтаксис для команды расширения FUZZY, а также некоторый другой код. Кроме того, FUZZY обеспечивает более универсальные способы проведения сопоставлений. Чтобы вывести справку по синтаксису для FUZZY, введите следующий синтаксис:

FUZZY /HELP.

У команды FUZZY есть несколько возможностей, не поддерживаемых в этом диалоговом окне, включая использование отдельных наборов данных наблюдений и контрольных значений в качестве входных данных (выполняется сопоставление для набора переменных без промежуточной логистической регрессии и сопоставление нескольких контрольных значений с каждым наблюдением). Для вычисления нечеткого расстояния можно также использовать пользовательские функции, написанные на языке Python.

Технические требования

Для выполнения этой команды требуется Python Essentials и IBM SPSS Statistics версии 20 или более поздней. Для использования единого набора данных может потребоваться Statistics версии 21 или оперативное исправление для версии 20.

Требуется команда расширения FUZZY версии 1.3.0 или новее. Эта версия новее поставляемой со Statistics 21.0.0.0. Python Essentials можно скачать с сайта сообщества SPSS по адресу www.ibm.com/developerworks/spssdevcentral. Последнюю версию FUZZY можно найти в собрании команд расширения.
