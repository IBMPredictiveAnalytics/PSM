Messa in corrispondenza punteggi propensione

Questa procedura mette in corrispondenza i record del caso con record di controllo simili contenuti in un singolo dataset. Esegue prima una regressione logistica con la variabile di gruppo di casi/controllo come variabile dipendente. Quindi, seleziona una corrispondenza per ciascun caso dal gruppo di controllo in base al punteggio propensione dalla regressione logistica. Il punteggio è una stima della probabilità di appartenenza nel gruppo di casi.

La procedura produce e attiva un nuovo dataset di casi e controlli messi in corrispondenza.

Indicatore gruppo Selezionare la variabile che definisce se un record è nel gruppo di casi o di controllo. Il valore della variabile uno indica un caso e il valore zero indica un controllo.

Predittori Selezionare le variabili da utilizzare nella regressione logistica per modellare l'appartenenza a casi/controlli.

Nome per variabile propensione Immettere un nome per la variabile che conterrà il punteggio (propensione) dalla regressione logistica. Questa variabile verrà utilizzata per la corrispondenza.

• Il nome della variabile non deve essere già in uso.

Tolleranza delle corrispondenze Specificare la tolleranza per il punteggio nei controlli e nei casi corrispondenti. Un controllo è idoneo per la corrispondenza a un caso se il valore assoluto della differenze nei punteggi di propensione è inferiore o uguale a questo valore. Il valore 0 indica solo corrispondenze esatte mentre il valore 1 indica che qualsiasi controllo corrisponde a qualsiasi caso. Dei valori più piccoli producono delle corrispondenze più prossime, ma possono aumentare il numero di casi senza corrispondenza.

• Le tabelle di output prodotte da questa procedura mostrano la distribuzione del numero di corrispondenze idonee, tenuto conto della tolleranza e possono essere utili nella scelta di un valore di tolleranza.

ID caso Specificare una variabile identificativo del caso. Il valore ID caso per il controllo messo in corrispondenza sarà archiviato con ogni record di caso.

Nome variabile ID di corrispondenza Specificare un nome per la variabile che conterrà l'ID del record di controllo selezionato. Se non è possibile produrre una corrispondenza, il valore di variabile sarà mancante di sistema.

• Il nome della variabile non deve essere già in uso.

Nome dataset di output Specificare un nome per il dataset da creare. Il dataset conterrà tutti i casi e i controlli con cui era stata trovata una corrispondenza.

• Il nome del dataset non deve essere già in uso.

• L'output è un dataset, non un file permanente. È necessario salvare il dataset per creare un file effettivo.

Opzioni

Variabile per numero di casi idonei Facoltativamente, immettere un nome di variabile per registrare la dimensione del pool di controlli, ossia il numero di casi idonei, per ogni record di caso. Questo è il numero di controlli che soddisfa i criteri di messa a confronto dopo che le corrispondenze per i record di caso precedenti sono state rimosse, in caso di campionamento senza sostituzione.

• Il nome della variabile non deve essere già in uso

Campionamento I casi di controllo possono essere estratti con o senza sostituzione.

Dai la priorità alle corrispondenze esatte Selezionare questa casella per fare in modo che l'algoritmo di messa in corrispondenza provi prima una corrispondenza esatta per poi tentare una corrispondenza approssimativa.

Massimizza prestazioni esecuzione Selezionare questa casella per massimizzare le prestazioni nella selezione delle corrispondenze - massima velocità e minimo utilizzo della memoria. Per dataset di grandi dimensioni, questa casella deve essere selezionata. Quando viene utilizzata, c'è una probabilità un po' più elevata che un caso non venga rilevato come corrispondente con un controllo.

• Non è possibile specificare sia la priorità esatta che la minimizzazione della memoria.

Rendi casuale l'ordine dei casi quando si estraggono le corrispondenze Selezionare questa casella per estrarre le corrispondenze per i record di caso in ordine casuale. Altrimenti, i record di caso vengono elaborati nell'ordine in cui ricorrono nel file. Ciò potrebbe determinare un numero di controlli inferiore da cui effettuare la selezione successivamente nel file. Se i record di caso hanno un ordine sistematico, l'elaborazione nell'ordine del file potrebbe introdurre un certo grado di distorsione.

• Questo non ha alcun effetto quando l'opzione Massimizza prestazioni esecuzione è selezionata.

Seme per numeri casuali Se si desidera poter riprodurre le corrispondenze esattamente in una futura esecuzione con gli stessi dati, è possibile impostare un valore del seme per i numeri casuali utilizzati quando si effettua la selezione da un insieme in modo casuale. Questo seme non è correlato al seme per numeri casuali generale di SPSS Statistics.

Requisiti

Questo comando richiede Python Essentials e almeno la versione 20 di IBM SPSS Statistics. Per l'utilizzo di dataset singoli, potrebbe essere richiesto Statistics versione 21 oppure una hot fix per la versione 20.
