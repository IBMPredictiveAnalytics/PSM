Correspondência de Escore de Propensão

Este procedimento corresponde registros de caso com registros de controle semelhantes contidos em um único conjunto de dados. Ele primeiro executa uma regressão logística com a variável de grupo caso/controle como a variável dependente. Em seguida, ele seleciona uma correspondência para cada caso a partir do grupo de controle baseado no escore de propensão da regressão logística. O escore é uma estimativa da probabilidade de associação no grupo de caso.

O procedimento produz e ativa um novo conjunto de dados de casos e controles correspondidos.

Indicador de Grupo Selecione a variável que define se um registro está no grupo de caso ou de controle. Um valor de variável de um indica um caso e um valor de zero indica um controle.

Preditores Selecione as variáveis a serem usadas na regressão logística para modelar a associação caso/controle.

Nome para a Variável de Propensão Insira um nome para a variável para conter o escore (propensão) da regressão logística. Essa variável será utilizada para correspondência.

• O nome da variável não deve estar em uso.

Tolerância de Correspondência Especifique a tolerância para o escore em casos e controles de correspondência. Um controle é elegível para corresponder a um caso se o valor absoluto da diferença nos escores de propensão for menor ou igual a esse valor. Um valor 0 significa correspondências exatas apenas enquanto um valor de 1 significar que qualquer controle poderia corresponder a qualquer caso. Valores menores produzem correspondências mais próximas, mas podem aumentar o número de casos sem correspondência.

• A tabela de saída produzida por este procedimento mostra a distribuição do número de correspondências elegíveis fornecida a tolerância e pode ser útil na escolha um valor de tolerância.

ID de Caso Especifique uma variável identificadora de caso. O valor do ID de Caso para o controle correspondido será armazenado com cada registro do caso.

Nome de Variáveis de ID de Correspondência Especifique um nome para a variável que conterá o ID do registro de controle selecionado. Se uma correspondência não puder ser produzida, o valor da variável será omisso do sistema.

• O nome da variável não deve estar em uso.

Nome do Conjunto de Dados de Saída Especifique um nome para o conjunto de dados a ser criado. O conjunto de dados irá reter todos os casos e os controles que foram correspondidos com eles.

• O nome do conjunto de dados não deve já estar em uso.

• A saída é um conjunto de dados, não um arquivo permanente. Você deve salvar o conjunto de dados para criar um arquivo real.

Opções

Variável para Número de Casos Elegíveis Opcionalmente, insira um nome de variável para registrar o tamanho do conjunto de controle, isto é, o número de casos elegíveis para cada registro de caso. Este é o número de controles que atendam aos critérios de correspondência depois de correspondências para os registros de caso anteriores serem removidas, se amostragem, sem reposição.

• O nome da variável não deve estar em uso

Amostragem Os casos de controle podem ser desenhados com ou sem reposição.

Dar prioridade a correspondência exata Marque esta caixa para que o algoritmo de correspondência tente primeiro uma correspondência exata antes de tentar uma correspondência difusa.

Maximizar o desempenho de execução Marque esta caixa para maximizar o desempenho de execução - velocidade máxima e uso de memória mínimo. Para grandes conjuntos de dados, esta caixa deve ser marcada. Quando ela é usada, há uma chance um pouco maior de que um caso não localizado seja correspondido com um controle.

• Você pode especificar prioridade exata e minimizar memória.

Escolher ordem de caso aleatoriamente ao traçar correspondências Marque para desenhar correspondências para os registros de caso em ordem aleatória. Caso contrário, os registros de caso são processados na ordem na qual eles ocorrem no arquivo. Isso poderia resultar em menos controles para selecionar mais tarde no arquivo. Se os registros de caso tiverem uma ordem sistemática, o processamento na ordem do arquivo pode introduzir algum viés.

• Isso não tem efeito quando a opção Maximizar o desempenho de execução está marcada.

Semente de Número Aleatório Para ter a capacidade de reproduzir as correspondências exatamente em uma futura execução com os mesmos dados, configure um valor semente para os números aleatórios usados na escolha aleatória em um conjunto. Essa semente não está relacionada com a semente de número aleatório geral do SPSS Statistics.

Recursos adicionais

Este diálogo gera sintaxe para o comando de extensão FUZZY bem como alguns outros códigos. FUZZY também fornece maneiras mais gerais para fazer correspondência. Para exibir ajuda de sintaxe para FUZZY, execute a seguinte sintaxe:

FUZZY /HELP.

FUZZY possui vários recursos não suportados neste diálogo, incluindo o uso de conjuntos de dados de caso e de conjunto separados como entrada, correspondência em um conjunto de variáveis sem a regressão logística intermediária e correspondência de vários controles com cada caso. Funções customizadas gravadas em Python também podem ser usadas para calcular a distância difusa.

Requisitos

Este comando requer o Python Essentials e pelo menos a versão 20 do IBM SPSS Statistics. Para uso de conjunto de dados único, o Statistics versão 21 ou um hot fix para a versão 20 pode ser necessário.

Ele requer pelo menos a versão 1.3.0 do comando de extensão FUZZY. Ele é mais recente que a versão fornecida com o Statistics 21.0.0.0 Você pode fazer download do Python Essentials no website da SPSS Community em www.ibm.com/developerworks/spssdevcentral. A versão mais recente de FUZZY pode ser localizada na coleção de Comandos de Extensão.
